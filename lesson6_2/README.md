## Теоретична довідка

**DNS (Domain Name System)** — ієрархічна розподілена система для перетворення доменних імен на IP-адреси.

**BIND (Berkeley Internet Name Domain)** — найпоширеніше програмне забезпечення DNS-сервера, що обслуговує значну частину доменних імен в Інтернеті.

> 💡 **Чому `.lab`, а не `.com`?**  
> Домен `tsa233.lab` використовує неіснуючий у публічному інтернеті TLD `.lab`. Це гарантує, що запити до цього домену **ніколи не вийдуть за межі лабораторної мережі** і не перетинатимуться з реальними сайтами. Для навчальних стендів завжди використовуйте `.lab`, `.test` або `.internal`.

### Типи DNS-записів, що використовуються в роботі:

| Тип запису | Призначення | Приклад |
|-----------|-------------|---------|
| `SOA` | Start of Authority — параметри зони | Серійний номер, TTL |
| `NS` | Name Server — вказує DNS-сервер зони | `ns1.tsa233.lab.` |
| `A` | Address — IPv4-адреса хоста | `tsa233.lab. → 192.168.1.100` |
| `CNAME` | Canonical Name — псевдонім | `www → tsa233.lab.` |
| `PTR` | Pointer — зворотна відповідність | `100 → tsa233.lab.` |

### Типи DNS-серверів:

- **Authoritative (авторитетний)** — зберігає оригінальні записи зони
- **Recursive (рекурсивний)** — виконує запити від клієнтів, кешує відповіді
- **Forwarding** — перенаправляє запити до іншого сервера

## 1. Встановлення BIND9

### 1.1 Оновлення пакетів та встановлення

```bash
# Оновити список пакетів
sudo apt update && sudo apt upgrade -y

# Встановити BIND9 та допоміжні утиліти
sudo apt install -y bind9 bind9utils bind9-doc dnsutils

# Перевірити статус служби
sudo systemctl status bind9
```

### 1.2 Увімкнути автозапуск

```bash
sudo systemctl enable bind9
```

### 1.3 Перевірити встановлену версію

```bash

```

## 2. Налаштування BIND сервера

### 2.1 Структура конфігураційних файлів

```
/etc/bind/
├── named.conf               # Головний файл (підключає інші)
├── named.conf.options       # Параметри роботи сервера
├── named.conf.local         # Локальні зони (редагуємо тут)
├── named.conf.default-zones # Стандартні зони (не чіпаємо)
└── zones/                   # Директорія для файлів зон (створимо)
```

### 2.2 Налаштування параметрів сервера

Відредагуйте файл `/etc/bind/named.conf.options`:

```bash
sudo nano /etc/bind/named.conf.options
```

```conf
options {
    directory "/var/cache/bind";

    // Дозволити рекурсивні запити лише з локальної мережі
    allow-recursion {
        localhost;
        192.168.1.0/24;
    };

    // Форвардинг зовнішніх запитів на DNS Google
    forwarders {
        8.8.8.8;
        8.8.4.4;
    };
    forward only;

    // Прослуховувати всі інтерфейси
    listen-on { any; };

    // Вимкнути IPv6 (якщо не використовується)
    listen-on-v6 { none; };

    // Дозволити запити від локальної мережі
    allow-query {
        localhost;
        192.168.1.0/24;
    };

    dnssec-validation no;
};
```

### 2.3 Перевірити синтаксис конфігурації

```bash
sudo named-checkconf
```

> ✅ Якщо немає жодного виводу — синтаксис правильний.

## 3. Створення зони tsa233.lab

### 3.1 Оголосити зону в named.conf.local

```bash
sudo nano /etc/bind/named.conf.local
```

Додати:

```conf
// Пряма зона (ім'я → IP)
zone "tsa233.lab" {
    type master;
    file "/etc/bind/zones/db.tsa233.lab";
};

// Зворотна зона (IP → ім'я)
zone "1.168.192.in-addr.arpa" {
    type master;
    file "/etc/bind/zones/db.192.168.1";
};
```

### 3.2 Створити директорію для зон

```bash
sudo mkdir -p /etc/bind/zones
```

### 3.3 Створити файл прямої зони

```bash
sudo nano /etc/bind/zones/db.tsa233.lab
```

```dns
;
; Файл прямої зони для tsa233.lab
;
$TTL    604800
@   IN  SOA ns1.tsa233.lab. admin.tsa233.lab. (
                  2024010101  ; Serial (формат: YYYYMMDDNN)
                  3600        ; Refresh (1 год)
                  1800        ; Retry (30 хв)
                  604800      ; Expire (1 тиждень)
                  86400 )     ; Negative Cache TTL (1 день)

; Записи Name Server
@       IN  NS      ns1.tsa233.lab.

; A-записи (ім'я → IP)
ns1     IN  A       192.168.1.10    ; DNS-сервер (Ubuntu VM)
@       IN  A       192.168.1.100   ; Основний сайт (ПК викладача)
www     IN  A       192.168.1.100   ; www псевдонім

; CNAME (псевдонім)
web     IN  CNAME   tsa233.lab.
```

### 3.4 Створити файл зворотної зони

```bash
sudo nano /etc/bind/zones/db.192.168.1
```

```dns
;
; Файл зворотної зони для 192.168.1.0/24
;
$TTL    604800
@   IN  SOA ns1.tsa233.lab. admin.tsa233.lab. (
                  2024010101  ; Serial
                  3600        ; Refresh
                  1800        ; Retry
                  604800      ; Expire
                  86400 )     ; Negative Cache TTL

; Name Server
@   IN  NS  ns1.tsa233.lab.

; PTR-записи (останній октет IP → повне ім'я)
10  IN  PTR ns1.tsa233.lab.     ; 192.168.1.10
100 IN  PTR tsa233.lab.         ; 192.168.1.100
```

### 3.5 Перевірити синтаксис файлів зон

```bash
# Перевірка прямої зони
sudo named-checkzone tsa233.lab /etc/bind/zones/db.tsa233.lab

# Перевірка зворотної зони
sudo named-checkzone 1.168.192.in-addr.arpa /etc/bind/zones/db.192.168.1
```

> ✅ Очікуваний вивід: `zone tsa233.lab/IN: loaded serial 2024010101` та `OK`

### 3.6 Встановити правильні права доступу

```bash
sudo chown -R bind:bind /etc/bind/zones/
sudo chmod 644 /etc/bind/zones/*
```

### 3.7 Перезапустити BIND9

```bash
sudo systemctl restart bind9
sudo systemctl status bind9
```

## 4. Налаштування клієнтів Windows

### 4.1 Налаштування DNS через графічний інтерфейс

1. Натиснути `Win + R` → ввести `ncpa.cpl` → Enter
2. Правою кнопкою на активний мережевий адаптер → **Властивості**
3. Вибрати **Протокол Інтернету версії 4 (TCP/IPv4)** → **Властивості**
4. Встановити **«Використовувати такі адреси DNS-серверів»**:
   - Основний: `192.168.1.10` *(IP Ubuntu DNS-сервера)*
   - Альтернативний: `8.8.8.8` *(щоб не втратити доступ до інтернету)*
5. Натиснути **ОК**

### 4.2 Налаштування через PowerShell (від Адміністратора)

```powershell
# Визначити ім'я мережевого інтерфейсу
Get-NetAdapter

# Встановити DNS (замінити "Ethernet" на ваш інтерфейс)
Set-DnsClientServerAddress -InterfaceAlias "Ethernet" -ServerAddresses ("192.168.1.10","8.8.8.8")

# Очистити кеш DNS
Clear-DnsClientCache

# Перевірити налаштування
Get-DnsClientServerAddress
```

### 4.3 Перевірка на клієнті

```cmd
:: Перевірити розпізнавання домену
nslookup tsa233.lab
nslookup www.tsa233.lab

:: Пінг за доменним ім'ям
ping tsa233.lab

:: Детальний DNS-запит напряму до нашого сервера
nslookup -type=A tsa233.lab 192.168.1.10
```

## 5. Перевірка роботи

### На DNS-сервері (Ubuntu):

```bash
# Перевірити логи BIND
sudo journalctl -u bind9 -f

# Тестові запити з сервера
dig @localhost tsa233.lab
dig @localhost www.tsa233.lab
dig @localhost -x 192.168.1.100

# Перевірити відкриті порти
sudo ss -tulnp | grep named
```

### На клієнті (Windows):

```cmd
nslookup tsa233.lab
nslookup www.tsa233.lab
nslookup -type=NS tsa233.lab
nslookup -type=SOA tsa233.lab
```

Відкрийте `http://tsa233.lab` — має відкритися сайт, запущений на ПК викладача у Docker.

## Завдання на самопідготовку

> Виконати після заняття. Результати надати у вигляді скриншотів або конфігураційних файлів.

### Завдання 1 — Базове (обов'язкове)

Додати до зони `tsa233.lab` наступні записи:
- `mail.tsa233.lab` → `192.168.1.20` (тип `A`)
- `ftp.tsa233.lab` → `192.168.1.30` (тип `A`)
- Запис `MX` для домену з пріоритетом `10`, що вказує на `mail.tsa233.lab`

Перевірити: `nslookup -type=MX tsa233.lab`

---

### Завдання 2 — Середнє

Налаштувати **другий DNS-сервер** (slave/secondary):

1. На другій віртуальній машині встановити BIND9
2. Налаштувати зону `tsa233.lab` як `slave` з master-сервером `192.168.1.10`
3. Переконатися, що зона автоматично синхронізується після зміни Serial

```conf
// Конфігурація slave-зони на другому сервері
zone "tsa233.lab" {
    type slave;
    masters { 192.168.1.10; };
    file "/var/cache/bind/db.tsa233.lab.slave";
};
```

---

### Завдання 3 — Підвищеної складності

Налаштувати **split-horizon DNS** (різні відповіді для різних клієнтів):

- Клієнти мережі `192.168.1.0/24`: `tsa233.lab` → `192.168.1.100`
- Клієнти мережі `10.0.0.0/8`: `tsa233.lab` → `10.0.0.100`

Використати директиву `view` у конфігурації BIND.

## Корисні команди

### Управління службою

```bash
sudo systemctl start bind9      # Запустити

sudo systemctl restart bind9    # Перезапустити
```

### Діагностика

```bash
# Перевірити всі конфігураційні файли разом із зонами

# Логи в реальному часі

# Перевірити, чи слухає порт 53
```
